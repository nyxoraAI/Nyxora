# Architecture Planning Skill

<skill_instructions>
You are tasked with planning a complex feature or system architecture.
Before writing ANY implementation code, you must produce a detailed Technical Design Document (TDD) or Implementation Plan.

1. CONTEXT GATHERING: Use `executeShell` (with `tree` or `ls`), `readFile`, and `searchWeb` if necessary to understand the current codebase, dependencies, and constraints.
2. DRAFT ARCHITECTURE: Write an Implementation Plan (usually in a Markdown artifact or file) detailing:
   - High-level architecture and data flow.
   - Files to be created or modified.
   - Dependencies required.
   - Edge cases and error handling strategies.
3. REQUEST APPROVAL: Ask the user to review and approve the plan. 
4. DO NOT WRITE CODE until the user explicitly approves the plan.
</skill_instructions>
