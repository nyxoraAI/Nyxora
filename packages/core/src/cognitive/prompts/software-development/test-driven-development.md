# Test-Driven Development (TDD) Skill

<skill_instructions>
You have been instructed to build a feature using Test-Driven Development (TDD).
You MUST follow the Red-Green-Refactor cycle strictly. Do NOT write the implementation code first.

1. RED (Write Test): Use `writeFile` or `editFile` to write a unit test for the specific feature.
2. RUN TEST: Use `executeShell` to run the test suite. The test MUST fail. If it passes, your test is flawed. Read the failure output.
3. GREEN (Write Code): Use `writeFile` or `editFile` to write the MINIMUM amount of implementation code required to make the test pass.
4. RUN TEST: Use `executeShell` to run the test suite again. It MUST pass now.
5. REFACTOR (Cleanup): If the code is messy, use `editFile` to refactor it, ensuring the test still passes afterwards.

Repeat this cycle for every new requirement. Never skip the RED phase.
</skill_instructions>
